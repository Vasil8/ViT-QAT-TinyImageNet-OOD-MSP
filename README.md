# ViT-QAT-TinyImageNet-OOD-MSP
ViT QAT (torchao W4A8, hybrid) on Tiny-ImageNet: Tiny-ImageNet-C corruptions + OOD detection (MSP) vs CIFAR-10/SVHN.

Репозиторий с Jupyter-блокнотом [`ViT_QAT_OOD_robustness.ipynb`](ViT_QAT_OOD_robustness.ipynb), в котором собран **единый конвейер экспериментов**: дообучение Vision Transformer на Tiny-ImageNet-200, квантизация с обучением (QAT) через **torchao**, оценка на **искажённых данных** (Tiny-ImageNet-C) и **OOD-метрики** по методу **MSP** (Maximum Softmax Probability).

## Конвейер экспериментов

1. **Данные и модель**  
   - Загрузка и подготовка **Tiny-ImageNet-200** (скачивание архива, разбор `train` / `val`).  
   - Базовая модель из **timm**: `vit_small_patch16_224.augreg_in21k_ft_in1k`.

2. **Три варианта обучения / квантизации** (последовательно в `main()`):  
   - **FP32** — полное дообучение в плавающей точке, контрольная точка для точности и для относительных метрик коррупций.  
   - **Full QAT (W4A8)** — подготовка графа под QAT, дообучение, конвертация: активации **int8** (динамика), веса **int4** (`Int8DynActInt4WeightQATQuantizer`).  
   - **Hybrid QAT** — QAT применяется только к выбранному диапазону блоков (остальные слои остаются в FP32), что даёт компромисс размер/качество.

3. **Оценка на чистых и искажённых ID-данных**  
   - Точность на чистой валидации Tiny-ImageNet.  
   - **Tiny-ImageNet-C**: для каждой коррупции и уровня **severity** (по умолчанию 1–5) изображение проходит цепочку **PIL → `imagecorruptions` → val-трансформ timm** (как в коде: согласованный с классификацией препроцесс).  
   - Агрегируется средняя точность по сетке коррупций; дополнительно строится **относительный показатель** `mCE_tinyinc_rel_fp32` — нормировка к FP32 на этом же прокси.

4. **OOD (MSP)**  
   - На **ID** (валидация Tiny-ImageNet) и на **OOD**: **CIFAR-10** и **SVHN** считаются softmax-скоры.  
   - Метрики: **AUROC**, **AUPR** с позитивным классом OOD (`1 - MSP`), **FPR при TPR = 0.95** по ROC; при необходимости — бутстреп-оценки (параметры в `CONFIG_SHARED`).

5. **Артефакты**  
   - Каталоги прогонов под `models_dir`, чекпойнты (`best_model_*.pth`), сводные таблицы (CSV), графики.  

## Запуск

- **Окружение:** Python 3.10+; для обучения желателен **GPU** (CUDA).  
- **Зависимости:**

```bash
pip install -r requirements.txt
```

- Откройте [`ViT_QAT_OOD_robustness.ipynb`](ViT_QAT_OOD_robustness.ipynb) и выполните ячейки сверху вниз. Первая ячейка кода ставит пакеты через `pip` (удобно для Colab); при локальной установке можно пропустить строку `!pip install` после `pip install -r requirements.txt`.
